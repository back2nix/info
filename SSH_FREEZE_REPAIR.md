# Диагностика и решение проблемы зависания системы при SSH

## 🔴 Проблема
При запуске `ssh <host>` (особенно `ssh beget_k3s`) ноутбук полностью зависает:
1. Забивается вся оперативная память (RAM).
2. Процессор (CPU) нагружается на 100%.
3. Пропадает звук, замирает интерфейс (UI freeze).
4. Через некоторое время (когда соединение устанавливается или отваливается) система "отвисает".

## 🔍 Причины (Root Cause)

### 1. Рекурсивная петля прокси (Proxy Loop)
В системе установлены переменные окружения `HTTP_PROXY` и `HTTPS_PROXY`, указывающие на **sing-box** (порт 1083). 
- Когда запускается `ssh`, он пытается использовать эти прокси для своих нужд (проверка CRL, DNS и т.д.).
- `sing-box` перехватывает этот трафик и пытается отправить его через свой исходящий канал (`outbound/ssh-out`).
- Если этот исходящий канал сам является SSH-соединением или использует те же системные прокси, возникает **бесконечная рекурсия**. Система плодит тысячи вложенных соединений в секунду, что мгновенно съедает всю память.

### 2. Конфликт SSH-агентов (GNOME Keyring / GCR)
Переменная `SSH_AUTH_SOCK` указывает на `/run/user/1000/gcr/ssh` (GCR-агент из GNOME).
- В NixOS и GNOME 45+ этот агент часто уходит в плотный цикл вычислений при попытке подписать запрос ключом `id_rsa`, если есть конфликт версий или блокировка секретов.
- Из-за высокого приоритета системного процесса это вызывает прерывание звукового потока и "фриз" графической оболочки.

---

## ✅ Решение (Fix)

Чтобы изолировать SSH от "токсичного" окружения, необходимо добавить следующие параметры в `~/.ssh/config`.

### 1. Глобальная защита (для всех хостов)
Добавь это в начало файла в секцию `Host *`:

```ssh
Host *
    # Запрещает SSH использовать системные прокси из ENV (убирает петлю в sing-box)
    ProxyCommand none
    # Отключает глючный GNOME Keyring / GCR агент
    IdentityAgent none
    # Форсирует быстрый и стабильный алгоритм (исключает тяжелый Post-Quantum KEX)
    KexAlgorithms curve25519-sha256
```

### 2. Индивидуальная настройка (для beget_k3s)
Убедись, что хост прописан корректно:

```ssh
Host beget_k3s
    HostName 109.69.21.245
    User root
    IdentityFile ~/.ssh/id_rsa
    # Использовать ТОЛЬКО указанный ключ (не перебирать ключи в агенте)
    IdentitiesOnly yes
```

## 🛠 Почему это работает
- **ProxyCommand none**: SSH подключается к серверу напрямую по TCP, игнорируя настройки прокси в системе. Рекурсия в `sing-box` невозможна.
- **IdentityAgent none**: SSH читает ключи напрямую с диска и сам запрашивает пароль в терминале, не обращаясь к зависающему `gnome-keyring`.
- **KexAlgorithms**: Алгоритм `curve25519` работает с маленькими пакетами и не нагружает CPU так сильно, как новые "пост-квантовые" методы.
